Bachelor Thesis — DTU Compute

Normative Alignment of Recommender Systems at the User Level via Internal Label Shift

Supervisor: Jes Frellsen Completed: March 2026 Dataset: EB-NeRD (news recommendation)

The problem

News recommender systems are often optimized purely for engagement — clicks, watch time — which can quietly concentrate what people read into a narrow set of categories over time.

A prior method, NAILS, corrects for this by nudging a system's recommendations back toward a target mix of content categories. But it does this correction in aggregate — across all users and impressions at once. That raised a question worth testing: if the system looks balanced on average, does that mean any individual user's feed is actually balanced? Or can the system "average out" to look fine while some users still get a heavily skewed set of recommendations?

What I built

I extended NAILS to work at the level of a single recommendation event instead of only across the whole system. Rather than computing one global correction and applying it everywhere, my version computes a correction specific to each impression, based on what categories are actually available to that user at that moment.

I implemented both the original global method and my per-user extension, and tested both across three different ways of turning scores into an actual Top-10 list: a straightforward ranking, a stochastic sampling approach (Plackett-Luce), and a greedy re-ranking method inspired by prior calibration work (Steck).

What I found

Pooled, system-wide alignment and per-user alignment are not the same objective — a method can look like it's working when you average everything together, while individual users still see a skewed set of recommendations.
  • The original global method did improve alignment when measured in aggregate, exactly as designed.
  • But under a "typical user" view — averaging alignment per user rather than pooling everyone together — the global method barely moved the needle.
  • My per-user extension showed a clear, consistent improvement under that same typical-user view, for both of the divergence measures I tracked.
  • Stronger correction isn't free: pushing harder toward the target category mix increases how much the final list deviates from the original ranking (measured as regret and list overlap), so there's a real trade-off to manage rather than a setting to maximize.

Why it matters

The practical takeaway is that "does this alignment method work?" depends entirely on what you're measuring. If your goal is a healthy-looking system in aggregate, the original global method is fine. If your goal is making sure a typical individual's recommendations are actually balanced — which is arguably the more honest goal — the per-user approach does meaningfully better, at a controllable cost to ranking utility.
